# Phase 1: Data Model — SoWhat News App

**Feature**: `001-sowhat-news-mvp`
**Date**: 2026-04-17

There are no database entities in the MVP — all state lives either in the browser (`localStorage`) or as short-lived keys in Upstash Redis. This document captures the in-memory and over-the-wire shapes as the canonical model. The definitive runtime source of truth is `packages/shared/src/schemas/*.ts`; contract files in [`contracts/`](./contracts/) are generated from the same schemas.

## Entity: `AnonymousPersona` (client-only)

**Storage**: Browser `localStorage` under `sowhat.persona` via Zustand `persist`.
**PII status**: Zero. Values are from fixed enums; no free-text fields.

| Field | Type | Required | Validation |
|---|---|---|---|
| `careerIndustry` | `CareerIndustry` | ✅ | Must be one of enum values |
| `financialProfile` | `FinancialProfile` | ✅ | Must be one of enum values |
| `housingLifeStage` | `HousingLifeStage` | ✅ | Must be one of enum values |

**Zod schema** (`PersonaSchema`):

```ts
z.object({
  careerIndustry: z.enum([...CAREER_INDUSTRY_VALUES]),
  financialProfile: z.enum([...FINANCIAL_PROFILE_VALUES]),
  housingLifeStage: z.enum([...HOUSING_LIFE_STAGE_VALUES]),
})
```

**State machine** (client-only, managed by Zustand):

```
[Empty] --select any--> [Partial] --all 3 selected--> [Complete]
[Partial] --user cancels/clears--> [Empty]
[Complete] --user changes any anchor--> [Complete] (new combination)
```

Only `Complete` enables the "Generate My Daily Impact" button (FR-003).

## Entity: `AnonymousDeviceToken` (client-only opaque identifier)

**Storage**: Browser `localStorage` under `sowhat.deviceToken`.
**Shape**: `string` — a v4 UUID generated by `crypto.randomUUID()` on first visit.
**PII status**: None; random, unlinkable across devices.
**Lifecycle**: Created lazily on first API call; never rotated unless user clears storage.
**Transport**: Sent in the `X-Device-Token` HTTP header on every API call.

**Server handling**: The raw token is **never stored**. The Worker computes `sha256(token)` via WebCrypto and uses the hex digest as the Redis key suffix.

---

## Entity: `RateLimitRecord` (server-side, ephemeral)

**Storage**: Upstash Redis.
**Key**: `ratelimit:<sha256_hex_of_device_token>` (managed internally by `@upstash/ratelimit`).
**Value**: Managed by `@upstash/ratelimit` sliding-window implementation (typically `timestamp:count` tuples).
**TTL**: 48 hours, set on every write (spec clarification).
**PII status**: None — value is a hash; there is no reverse lookup to user identity because the server never sees the raw token except transiently in memory during the request.

**Invariants**:
- No reads or writes outside the `generate-impact` handler.
- Entries auto-expire; there is no administrative delete operation or dashboard.
- The Worker never logs the full hash; only the first 8 hex chars for debugging.

## Entity: `DailyNewsStory` (transient, server-side only)

**Source**: NewsAPI.org response items; never leaves the Worker unchanged.

| Field | Type | Required | Source |
|---|---|---|---|
| `originalHeadline` | `string` (1–300 chars) | ✅ | `article.title` |
| `originalUrl` | `string` (valid URL) | ✅ | `article.url` |
| `source` | `string` (1–120 chars) | ✅ | `article.source.name` |
| `publishedAt` | `string` (ISO-8601 UTC) | ✅ | `article.publishedAt` |

**Validation**: Items missing any required field or whose `publishedAt` is not within the UTC day (with the yesterday-fallback from research.md §4) are dropped.

## Entity: `ImpactNewsItem` (wire + render shape)

This is the unit of value the API emits and the UI renders. It is a **discriminated union** on `personalized` so partial success is explicit.

**Variant A — successfully personalized:**

| Field | Type | Constraint |
|---|---|---|
| `personalized` | `true` | literal |
| `originalHeadline` | `string` | 1–300 chars |
| `originalUrl` | `string` | valid URL |
| `source` | `string` | 1–120 chars |
| `impactHeadline` | `string` | 10–15 words (enforced post-generation); max 200 chars |
| `impactSummary` | `string` | 2 sentences (pragmatic check: `[.!?]` count in 1..3); max 500 chars |

**Variant B — fallback (personalization failed for this item):**

| Field | Type | Constraint |
|---|---|---|
| `personalized` | `false` | literal |
| `originalHeadline` | `string` | 1–300 chars |
| `originalUrl` | `string` | valid URL |
| `source` | `string` | 1–120 chars |
| `fallbackReason` | `"ai_timeout" \| "ai_invalid_output" \| "ai_error"` | enum |

**Zod schema sketch** (`ImpactNewsItemSchema`):

```ts
const Base = z.object({
  originalHeadline: z.string().min(1).max(300),
  originalUrl: z.string().url(),
  source: z.string().min(1).max(120),
});

const PersonalizedItem = Base.extend({
  personalized: z.literal(true),
  impactHeadline: z
    .string()
    .max(200)
    .refine(
      (s) => {
        const w = s.trim().split(/\s+/).length;
        return w >= 10 && w <= 15;
      },
      { message: "impactHeadline must be 10–15 words" },
    ),
  impactSummary: z
    .string()
    .max(500)
    .refine((s) => {
      const sentences = s.match(/[.!?]+(\s|$)/g)?.length ?? 0;
      return sentences >= 1 && sentences <= 3;
    }, { message: "impactSummary must be 1–3 sentences" }),
});

const FallbackItem = Base.extend({
  personalized: z.literal(false),
  fallbackReason: z.enum(["ai_timeout", "ai_invalid_output", "ai_error"]),
});

export const ImpactNewsItemSchema = z.discriminatedUnion("personalized", [
  PersonalizedItem,
  FallbackItem,
]);
```

**Why word/sentence ranges as refinements, not LLM `describe` alone**: The LLM `describe()` hints the model but doesn't guarantee the bound. Refinements run on server, log a warning when a generated item fails them, and coerce that item into the fallback variant (`fallbackReason: "ai_invalid_output"`) so the contract is never broken.

## Entity: `GenerateImpactRequest`

**Method**: `POST /api/generate-impact`
**Header**: `X-Device-Token: <uuid>` (required)
**Body** (Zod `GenerateImpactRequestSchema`):

```ts
z.object({
  persona: PersonaSchema,
});
```

**Validation failures** → 400 `validation_failed`.
**Missing header** → 400 `validation_failed` with `message: "X-Device-Token header required"`.

---

## Entity: `GenerateImpactResponse` (200)

```ts
z.object({
  generatedAt: z.string().datetime(),           // ISO-8601 UTC
  corpusDate: z.string().regex(/^\d{4}-\d{2}-\d{2}$/), // UTC day used
  newsItems: z.array(ImpactNewsItemSchema).min(1).max(5),
});
```

**Invariants**:
- `newsItems.length` is always between 1 and 5 inclusive (edge case in spec §Edge Cases).
- At least one item is either personalized or fallback; the response is never empty.
- `corpusDate` reflects the UTC day after yesterday-fallback (research.md §4).

---

## Entity: `ErrorResponse` (4xx/5xx)

**Zod** (`ErrorResponseSchema`):

```ts
z.discriminatedUnion("error", [
  z.object({
    error: z.literal("validation_failed"),
    message: z.string(),
    issues: z.array(z.object({ path: z.string(), message: z.string() })).optional(),
  }),
  z.object({
    error: z.literal("rate_limited"),
    message: z.string(),
    retryAfterSeconds: z.number().int().nonnegative(),
  }),
  z.object({
    error: z.literal("news_unavailable"),
    message: z.string(),
  }),
  z.object({
    error: z.literal("ai_unavailable"),
    message: z.string(),
  }),
  z.object({
    error: z.literal("internal_error"),
    message: z.string(),
    requestId: z.string().optional(),
  }),
]);
```

| `error` | HTTP status |
|---|---|
| `validation_failed` | 400 |
| `rate_limited` | 429 (also sets `Retry-After`) |
| `news_unavailable` | 502 |
| `ai_unavailable` | 502 |
| `internal_error` | 500 |

Messages are user-facing copy; technical details (stack, upstream status codes) go only to Worker logs tagged with `requestId`.

## Relationships

```
AnonymousPersona ───► (wire) GenerateImpactRequest.persona
AnonymousDeviceToken ───► (header) X-Device-Token ───► sha256 ───► RateLimitRecord (Redis)
DailyNewsStory ─(AI transform)─► ImpactNewsItem (personalized | fallback)
ImpactNewsItem[] ───► GenerateImpactResponse.newsItems ───► UI ImpactCard
```

**No foreign keys anywhere** — there is no database. The only server-side persisted artifact is the rate-limit record, which is tied to the hashed token and auto-expires.

## Mapping to spec requirements

| Requirement | Model element |
|---|---|
| FR-001 three inputs | `AnonymousPersona` with 3 required enum fields |
| FR-002 browser persistence, no PII | `localStorage` + Zustand `persist` on `AnonymousPersona` |
| FR-003 block until complete | `PersonaSchema` rejects partial persona; UI disables button |
| FR-004 top 5 UTC-day headlines | `DailyNewsStory` filter + `GenerateImpactResponse.corpusDate` |
| FR-005 per-item content + fallback | `ImpactNewsItem` discriminated union |
| FR-006 loading state | Handled in UI; no data-model entity |
| FR-007 3/24h rate limit, hashed token, 48h expiry | `AnonymousDeviceToken` + `RateLimitRecord` with TTL |
| FR-008 rate-limit message | `ErrorResponse.rate_limited` with `retryAfterSeconds` |
| FR-009 user-friendly errors | `ErrorResponse` discriminated union |
| FR-011 "Why this matters to you" badge + source footer | Rendered from `ImpactNewsItem` fields |
| FR-012 persona editable | Zustand store replacement; no server state to migrate |
| FR-013 WCAG 2.1 AA | UI-level concern; no data-model entity |
